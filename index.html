Version Control ⚙️

What is Version Control?

Version control, also known as source control, is a way to track and manage changes to software code over time.

How does it work?

Basic workflow: Commit, push, pull

Why is it important?

Version control is important because it allows you to keep track of changes to your code, collaborate with others, and revert to previous versions if needed.

Imagine working on a large project with multiple team members. How do you keep track of everyone's changes without overwriting each other's work?

This is where Git comes in....

What is Git?

Git is a distributed version control system that allows you to track changes to your code, collaborate with others, and manage your projects.

How does Git work?

Git works by creating snapshots of your code at different points in time. These snapshots are called commits.

Each commit contains a reference to the previous commit, creating a history of changes to your code.

What is a commit?

A commit is a snapshot of your code at a specific point in time. It contains the changes you have made to your code since the last commit.

What is a repository?

A repository, or repo, is a collection of files and folders that are tracked by Git. It contains the entire history of your project, including all commits and branches.

What is a branch?

A branch is a separate line of development in Git. It allows you to work on new features or experiment with changes without affecting the main codebase.

Why should I use Git?

Allows you to track changes to your code over time

Enables you to collaborate with others on the same codebase

You can easily revert to a previous version of your code or experiment with new features without affecting the main codebase

How to install Git?

  • Download Git from the official website
  • Run the installer
  • Configure Git
Congratulations! You have successfully installed Git on your local machine. You are now ready to start using Git for your projects.

How to Configure Git

Configuring your name and email address on your local machine is an essential step in setting up Git. 


							git config --global user.name "Your Name"
							git config --global user.email "your-email@example.com"

Now that we have finished setting up your username and email for Git and GitHub, let's verify that everything is configured correctly. To do this, use the following command:


							git config --list

Most used Git Commands 


							git init
							git add .
							git commit -m "Initial commit"
							git push origin master
							git pull origin master

But where is my code stored?

GitHub

GitHub is a web-based platform that allows you to host your code, collaborate with others, and manage your projects.

Why should I use GitHub?

GitHub provides several benefits:

  • Code hosting
  • Collaboration
  • Project management
  • Creating your own website using GitHub Pages, a static site hosting service.

How do I get started with GitHub?

Getting started with GitHub is easy:

  1. Create a GitHub account
  2. Create a repository
  3. Push your code to GitHub
  4. Collaborate with others

What are some common GitHub features?

Some common GitHub features include:

  • Issues - to track bugs and feature requests
  • Pull requests - to propose changes to a repository
  • Projects - to organize and prioritize work
  • Actions - to automate workflows

GitHub Cli

GitHub CLI brings GitHub to your terminal. It reduces context switching, helps you focus, and enables you to more easily script and create your own workflows.

How to install GitHub CLI

  1. Download GitHub CLI from the official website
  2. Run the installer
  3. Authenticate with GitHub

How to authenticate with GitHub CLI

Authenticating with GitHub CLI allows you to interact with GitHub from your terminal. To authenticate with GitHub CLI, use the following command:


						gh auth login

And anyone can contribute to it!!

Why should I contribute to Open Source?

apart from getting a cool badge like this

How do I get started with Open Source?

Getting started with open source is easy:
